# 단기계획 마무리 및 중기계획 준비 2026-03-17

## 단기계획 완료 상태

### 완료

- Search Console API 연결과 URL-prefix 속성 검증 완료
- GA4 AI 유입용 커스텀 차원 `ai_source`, `ai_referrer_host` 등록 완료
- AI/Tools 상위 글 `빠른 답`, `한눈에 고르기`, `이 글과 함께 보면 좋은 글` 구조 적용
- Search Console 기반 리라이트 1차 반영
- 주간 리뷰와 Search Console 기회 문서 생성 완료

### 이번 라운드에서 마무리한 것

- 홈과 백준 허브에서 다음 클릭을 더 잘 만들도록 허브형 동선 보강
- CTR이 낮거나 순위 `5~15` 구간에 들어온 레거시 랜딩 페이지 메타/내부링크 보강
- AI 허브를 중기 운영용 클러스터 허브 구조로 확장

## 중기계획 준비 상태

### 새로 만든 클러스터 허브

- `/ai/coding-tools/`
- `/ai/browsers-research/`
- `/ai/local-llm/`
- `/ai/creative/`
- `/ai/workflows/`

### 왜 이 구조로 갔는가

- AI 글이 늘수록 `한 페이지에 다 모아두는 허브`만으로는 다음 클릭 유도가 약해진다.
- 클러스터 허브가 있으면 비교 글 → 설치 글 → 개념 글 → 다음 허브 흐름이 자연스러워진다.
- Search Console이 AI/Tools 쪽을 더 인식하기 시작하면, 허브 단위로 어떤 클러스터가 성장하는지 따로 보기 쉬워진다.

## 중기계획 첫 실행 후보

1. 각 AI 클러스터 허브에 대표 썸네일과 추천 순서 더 보강
2. 클러스터별 대표 비교 글 1개씩 추가
3. 설치 글과 비교 글을 허브 기준으로 2차 내부링크 정리
4. Search Console에서 AI/Tools 랜딩이 잡히기 시작하면 클러스터 단위 CTR 개선

## 다음 재확인 시점

- Search Console 재수집 권장: `2026-03-20 ~ 2026-03-24`
- 목표:
  - 이번에 손본 레거시 랜딩의 CTR 변화 확인
  - AI/Tools 허브가 Search Console 랜딩으로 잡히기 시작했는지 확인
