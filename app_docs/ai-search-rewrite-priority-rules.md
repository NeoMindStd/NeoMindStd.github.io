# Search Console 기반 AI/Tools 리라이트 우선순위 규칙

## 목적

이 문서는 Search Console 데이터가 들어오기 시작했을 때 어떤 글부터 먼저 손볼지 빠르게 결정하기 위한 기준입니다.

핵심은 단순 조회수가 아니라 아래 두 가지를 함께 보는 것입니다.

- 노출은 있는데 클릭이 안 나오는 글
- 검색 결과에서 1페이지 하단~2페이지 초입에 걸쳐 있는 글

## 가장 먼저 볼 지표

1. `impressions`
2. `clicks`
3. `ctr`
4. `position`
5. 랜딩 페이지가 `AI`, `Tools`, `기타/AI` 축인지 여부

## 우선순위 규칙

### P1

아래 중 하나에 해당하면 가장 먼저 손봅니다.

- 노출 `100+`, CTR `2% 미만`
- 평균 순위 `4 ~ 15`, 노출 `50+`
- AI/Tools 핵심 허브 글인데 상위 검색 랜딩으로 잡히기 시작한 글

이 구간은 작은 제목/설명/첫 문단 수정만으로도 클릭과 순위가 같이 좋아질 가능성이 큽니다.

### P2

- 노출 `30 ~ 99`
- CTR은 아주 나쁘지 않지만 평균 순위가 `8 ~ 20`
- 비교 글, 설치 글, 오류 해결 글처럼 클릭 의도가 강한 주제

이 구간은 내부 링크, 허브 링크, 관련 글 배치 강화 효과가 큽니다.

### P3

- 노출 `30 미만`
- 최근 작성 직후라 데이터가 아직 적은 글
- 검색보다 공유/내부 유입이 먼저 붙는 글

이 구간은 바로 리라이트하기보다 데이터가 더 쌓일 때까지 관찰합니다.

## 글 유형별 액션

### 비교 글

- 제목에 비교 대상과 연도를 명확히 넣기
- 첫 3줄에 “누가 뭘 고르면 되는지” 바로 제시
- 표를 상단으로 끌어올리기

예시:

- `2026 프론티어 모델 비교`
- `AI 브라우저 비교`

### 설치 글

- 가장 추천하는 설치 경로를 맨 위에 배치
- 공식 사이트 / 다운로드 링크 / 지원 OS를 더 선명하게 정리
- 설치 후 첫 실행까지 캡처 또는 단계 요약 보강

예시:

- `Gemini CLI`
- `Claude Code`
- `Docker`
- `Ollama`

### 오류 해결 글

- 증상 문장을 제목에 그대로 넣기
- 원인과 해결 결과를 빠르게 제시
- 실패 사례와 주의점도 같이 적기

### 허브 글

- 관련 글 링크를 더 많이 심기보다 `다음 클릭`이 잘 나오는 3~6개만 엄선
- 비교 글, 설치 글, 개념 글이 섞이도록 연결

## 실제 수정 순서

1. 제목
2. 메타 설명 또는 첫 문단 요약
3. 첫 화면의 빠른 답 블록
4. 비교표 / 설치 요약 박스
5. 관련 글 링크

검색에서 클릭을 못 받는 글은 대개 위 5개에서 문제가 생깁니다.

## 지금 블로그 기준 추천 적용 순서

Search Console 데이터가 잡히면 아래 축을 먼저 봅니다.

1. `기타/AI` 비교 글
2. `기타/AI` 설치 글
3. `Tools` 설치 글
4. `창작 AI` 입문/비교 글
5. 오래된 문제풀이 글보다 이미 상위 랜딩으로 뜨는 허브 글

## 요약

Search Console 데이터가 생기면 모든 글을 한꺼번에 고치는 게 아니라, `노출은 있는데 클릭이 약한 글`, `순위 4~15에 있는 글`, `AI/Tools 핵심 글`부터 순서대로 손보는 것이 가장 효율적입니다.
