### 업무형 / 리서치형 / 에이전트형

| 흐름 | 먼저 볼 글 | 잘 맞는 상황 | 다음 단계 |
|:--|:--|:--|:--|
| 업무형 | [Gemini Workspace 가이드](/기타/gemini-workspace-guide/) | 문서, 표, 슬라이드, Drive 파일을 많이 다룰 때 | 프롬프트 가이드, 딥리서치 |
| 리서치형 | [Perplexity Labs](/기타/perplexity-labs-guide/), [딥리서치](/기타/what-is-deep-research/) | 자료 수집, 비교 조사, 보고서 초안이 많을 때 | AI 브라우저 허브, Eval |
| 에이전트형 | [컴퓨터 사용형 에이전트](/기타/computer-using-agent-guide/) | 브라우저나 도구 조작까지 맡기고 싶을 때 | MCP, A2A, Skill |
| 설계형 | [프롬프트 엔지니어링](/기타/prompt-engineering-practical-guide/), [MCP](/기타/what-is-mcp/) | 도구 연결 구조와 작업 명세를 같이 잡고 싶을 때 | RAG, Eval, 코딩 도구 허브 |

### 추천 순서

1. 문서 작업을 줄이고 싶다면 `Gemini Workspace`부터 봅니다.
2. 검색과 조사, 보고서 초안이 핵심이면 `딥리서치`, `Perplexity Labs`, `AI 브라우저 허브`를 같이 봅니다.
3. AI가 실제로 브라우저나 파일을 조작하는 흐름까지 보고 싶다면 `컴퓨터 사용형 에이전트`, `MCP`, `A2A`를 이어서 보는 편이 좋습니다.
4. 코딩 워크플로와 붙일 생각이 있다면 마지막에 `AI 코딩 도구 허브`로 넘어가면 연결이 자연스럽습니다.
