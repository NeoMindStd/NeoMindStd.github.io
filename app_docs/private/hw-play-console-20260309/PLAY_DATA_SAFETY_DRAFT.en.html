Play Console Data Safety Draft (HorrorWorld)

Last checked: 2026-03-09

This document is a practical draft for filling the Data safety form in Play Console. Final answers must be verified in Console before submission.

SDKs and features used by the app

  • Firebase Analytics
  • Google Mobile Ads SDK (AdMob)
  • Remote content fetch via HTTPS (GitHub raw URL)

Suggested declaration baseline

Does your app collect or share any user data?

  • Yes

Is all user data encrypted in transit?

  • Yes (HTTPS/TLS endpoints)

Can users request that their data be deleted?

  • Yes (request channel via developer contact email / privacy policy)

Data types likely applicable (confirm in Console)

  • App activity: app interactions, in-app events
  • Device or other IDs: advertising ID (AdMob), app instance identifiers
  • Diagnostics: crash/diagnostic signals (SDK-side)

Purposes likely applicable (confirm in Console)

  • Analytics
  • Advertising or marketing
  • App functionality (ad delivery and measurement)

Important consistency checks

  • Privacy policy URL must be publicly accessible and match actual behavior.
  • If AdMob is enabled, avoid submitting as "No data collected".
  • Data safety answers must match SDK usage in released artifact.

Related project evidence

  • Ads dependency: com.google.android.gms:play-services-ads:25.0.0
  • Firebase Analytics dependency is enabled
  • Merged release manifest includes AD_ID permission
  • targetSdkVersion=36, versionCode=20
