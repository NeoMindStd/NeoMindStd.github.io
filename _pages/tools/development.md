# 개발·협업 도구 허브

개발 환경을 세팅하거나 협업 흐름을 잡을 때 자주 쓰는 도구만 모았습니다.

### 한눈에 고르기

| 도구 | 잘 맞는 상황 | 대표 OS | 먼저 볼 글 |
|:--|:--|:--|:--|
| Docker | 개발 환경을 컨테이너로 통일하고 싶을 때 | Windows, macOS, Linux | [Docker 설치와 사용법](/tools/docker-guide/) |
| Python | 스크립트, 자동화, 데이터 작업을 시작할 때 | Windows, macOS, Linux | [Python 설치와 사용법](/tools/python-guide/) |
| OpenJDK / Oracle Java | JVM 기반 개발 환경을 맞춰야 할 때 | Windows, macOS, Linux | [OpenJDK 설치와 사용법](/tools/openjdk-temurin-guide/) |
| Git Bash | Windows에서 터미널 기반 Git 흐름이 필요할 때 | Windows | [Git Bash 설치와 사용법](/tools/git-bash-guide/) |
| GitKraken / SourceTree | GUI로 Git 브랜치와 이력을 보고 싶을 때 | Windows, macOS | [GitKraken 설치와 사용법](/tools/gitkraken-guide/) |
| Postman | API 테스트와 컬렉션 관리가 필요할 때 | Windows, macOS, Linux | [Postman 설치와 사용법](/tools/postman-guide/) |
| Notepad++ / HxD | 빠른 편집이나 바이너리 확인이 필요할 때 | Windows | [Notepad++ 설치와 사용법](/tools/notepad-plus-plus-guide/) |

### 추천 순서

1. 개발 환경 전체를 먼저 정리하려면 `Docker`, `Python`, `Java/OpenJDK` 순으로 보는 편이 가장 무난합니다.
2. Git이 익숙하지 않다면 `Git Bash`로 기본 흐름을 잡고, 그 다음 `GitKraken`이나 `SourceTree` 같은 GUI 도구로 넘어가면 편합니다.
3. API 연동 작업이 많으면 `Postman`을 빠르게 붙여두는 편이 효율적입니다.
