# 문제/배경

문제풀이 글이 900개를 넘으면, 검색으로 들어온 사용자가 "다음 글"을 찾기 어려워집니다.  
한 글의 체류시간만 올리는 방식보다 **허브 페이지로 연결되는 내부 순환 구조**가 더 중요해졌습니다.

# 원인

1. 개별 포스트 단위로만 소비되고 시리즈 맥락이 약함
2. 카테고리/허브 진입 링크가 본문에 충분히 노출되지 않음
3. 문제풀이 외 프로젝트/기타 글로의 확장 동선이 부족함

# 적용 코드/설정

## 1) 허브 페이지 분리

아래 허브를 고정 축으로 두고, 주제별로 진입점을 분리했습니다.

- `/boj/`
- `/programmers/`
- `/projects/`
- `/etc/`
- `/series/`

## 2) 포스트 하단 공통 CTA

`_layouts/single.html`에서 포스트 카테고리에 따라 허브 링크가 자동 노출되도록 적용했습니다.

```liquid
{% if page.categories contains "문제풀이" %}
  <li><a href="/boj/">백준 문제풀이 허브</a></li>
  <li><a href="/programmers/">프로그래머스 문제풀이 허브</a></li>
{% endif %}
```

## 3) 허브 소개문 검색형 문장으로 정리

`index`, `series`, `boj`, `programmers`, `projects`, `etc`의 상단 소개 문구를  
사용자 검색 의도(무엇을 얻을 수 있는지) 중심으로 통일했습니다.

# 검증 결과

- 개별 문제 유입 후 허브 재탐색 비율이 개선되는 구조를 확보
- 문제풀이 방문자를 프로젝트/운영 글로 확장할 수 있는 동선 확보
- 사이트맵/카테고리 페이지와의 연결성이 높아져 크롤링/탐색 효율 개선

# 체크리스트

- [ ] 각 카테고리 허브 URL이 상단/본문/하단 어디서든 접근 가능한가
- [ ] 문제풀이 글 하단에 다음 읽을거리 CTA가 자동 표시되는가
- [ ] 허브 페이지 상단 문구가 "주제 + 얻는 가치" 형태로 명확한가
- [ ] `/series/`를 전체 네비게이션 백본으로 사용하고 있는가
- [ ] SEO 목적 링크와 UX 목적 링크가 동시에 작동하는가
