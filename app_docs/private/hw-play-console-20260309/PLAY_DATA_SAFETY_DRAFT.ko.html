Play Console 데이터 보안 양식 초안 (HorrorWorld)

최종 확인: 2026-03-09

이 문서는 Play Console 데이터 보안 양식 작성용 실무 초안입니다. 실제 제출 전 콘솔에서 최종 항목을 반드시 재검증해야 합니다.

앱에서 사용하는 SDK 및 기능

  • Firebase Analytics
  • Google Mobile Ads SDK (AdMob)
  • HTTPS 기반 원격 콘텐츠 로드 (GitHub raw URL)

  • 예 (HTTPS/TLS 엔드포인트)

사용자가 데이터 삭제를 요청할 수 있습니까?

  • 예 (개발자 연락 이메일/개인정보처리방침을 통한 요청 채널)

적용 가능성이 높은 데이터 유형 (콘솔에서 확인 필요)

  • 앱 활동: 앱 상호작용, 인앱 이벤트
  • 기기 또는 기타 ID: 광고 ID(AdMob), 앱 인스턴스 식별자
  • 진단: 충돌/진단 신호(SDK 측 처리)

적용 가능성이 높은 처리 목적 (콘솔에서 확인 필요)

  • 분석
  • 광고 또는 마케팅
  • 앱 기능 제공(광고 송출 및 측정)

중요 정합성 점검

  • 개인정보처리방침 URL은 공개 접근 가능해야 하며 실제 앱 동작과 일치해야 합니다.
  • AdMob 사용 시 "수집 데이터 없음"으로 제출하면 안 됩니다.
  • 데이터 보안 양식 답변은 실제 릴리즈 산출물의 SDK 사용 상태와 일치해야 합니다.

관련 프로젝트 근거

  • 광고 의존성: com.google.android.gms:play-services-ads:25.0.0
  • Firebase Analytics 의존성 활성화
  • 릴리즈 머지 매니페스트에 AD_ID 권한 포함
  • targetSdkVersion=36, versionCode=20
