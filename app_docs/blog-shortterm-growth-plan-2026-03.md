# 블로그 단기 성장 실행 계획

## 목표

- AI/Tools 글을 블로그의 실제 유입 허브로 키운다.
- 최근 상승한 AI/Tools 포스트를 더 잘 읽히고, 더 많이 이어 읽히게 만든다.
- GA4/AdSense 중심 주간 리뷰를 고정해서 다음 액션을 숫자로 결정한다.
- Search Console은 권한 준비를 끝내서 다음 단계에서 검색어·평균순위를 합친다.

## 실행 범위

### 1. 콘텐츠 보강

- AI/Tools 상위 20개 글에 다음 구조를 넣는다.
  - 빠르게 보기 또는 한눈에 고르기
  - 이 글과 함께 보면 좋은 글
- 비교형, 설치형, 창작형, 로컬 LLM형 글을 우선 순위로 둔다.
- 이미 본문에 요약이 있는 글은 중복 추가하지 않는다.

### 2. 내부동선 개선

- 본문 끝 generic CTA만 두지 않고, 글 주제와 직접 연결되는 다음 읽을거리 링크를 넣는다.
- AI/Tools 허브와 개별 글 사이 이동을 더 짧게 만든다.

### 3. 측정 체계

- GA4/AdSense 스냅샷을 주간 리뷰로 바로 바꿔주는 스크립트를 추가한다.
- AI referral 이벤트는 이미 심어둔 상태라, 다음 주부터는 커스텀 차원 등록 여부와 함께 본다.
- Search Console은 현재 OAuth 스코프 부족 상태를 문서화하고 재인증 단계까지 정리한다.

### 4. 단기 수익 관점

- 광고 수를 더 늘리는 대신, 상위 랜딩 글의 품질과 다음 클릭을 먼저 높인다.
- 긴 AI/Tools 글 중심으로만 추가 실험을 검토하고, 모바일 UX를 해치지 않는 범위에서만 판단한다.

## 이번 턴에서 실제로 수행한 것

1. AI/Tools 상위 20개 글에 단기 성장용 메타 블록을 추가했다.
2. single 레이아웃에 `빠르게 보기`, `한눈에 고르기`, `이 글과 함께 보면 좋은 글` 블록 렌더링을 추가했다.
3. 주간 성장 리뷰 생성 스크립트를 추가했다.
4. Search Console 준비 상태 점검 스크립트를 추가했다.
5. 현재 OAuth 스코프 기준 Search Console 연결이 막혀 있다는 점을 문서로 남겼다.

## 다음 체크포인트

1. 다음 7일 뒤 주간 리뷰를 다시 생성한다.
2. AI/Tools 상위 랜딩 글이 실제로 늘었는지 확인한다.
3. Search Console 토큰 재발급 후 검색어 데이터를 합친다.
