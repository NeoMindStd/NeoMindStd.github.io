Google Play 복구 및 재출시 체크리스트 (2026-03-09 기준)

1) 계정 및 프로필

  • 개발자 프로필 정지 해제 이후 이 체크리스트를 사용합니다.
  • 조직 계정은 Play Console, 결제 프로필, 사업자 문서의 법인명/주소/국가를 일치시킵니다.

2) 현재 프로젝트 빌드 상태

  • targetSdkVersion: 36 (Android 15+ 요구사항 충족)
  • compileSdk: 36
  • minSdk: 24
  • versionCode: 20
  • versionName: 12.0.1
  • Google Mobile Ads SDK: com.google.android.gms:play-services-ads:25.0.0
  • Firebase Analytics 활성화
  • 릴리즈 빌드: R8 축소 + 리소스 축소 사용

3) Play Console에서 직접 완료해야 할 항목

A. 데이터 보안 양식 (필수)

  • 양식이 누락되면 업데이트/게시가 차단됩니다.
  • Firebase Analytics + AdMob 사용 중이므로 "수집 데이터 없음"으로 제출하면 안 됩니다.
  • SDK 데이터 처리, 처리 목적(분석/광고), 광고 ID 사용, 전송 중 암호화, 삭제 요청 채널을 검증 후 신고합니다.

B. 개인정보처리방침 URL (스토어 등록정보)

  • 공개적으로 접근 가능해야 합니다.
  • 실제 앱 동작(분석 + 광고 + 네트워크 콘텐츠 로드)과 일치해야 합니다.

C. 새 프로덕션 릴리즈 게시

  • 기존 프로덕션 경고(versionCode 18, targetSdk 29)는 더 높은 준수 버전을 프로덕션에 게시해야 해제됩니다.
  • 내부/비공개 테스트만으로는 프로덕션 가용성 경고가 해소되지 않을 수 있습니다.

4) 권장 롤아웃

  1. 내부 테스트 트랙
  2. 비공개 테스트 트랙
  3. 프로덕션 단계적 롤아웃 (5% -> 20% -> 100%)

5) 출시 후 확인

  • Play 정책 상태: 타겟 API 경고 해제, 구버전 GMA SDK 경고 해제, 데이터 보안 이슈 해제
  • Firebase 이벤트가 DebugView/Analytics 대시보드에 표시되는지 확인

Google Play Recovery and Re-Release Checklist (as of 2026-03-09)

1) Account and profile

  • Use this checklist after developer profile suspension is cleared.
  • For organization accounts, keep legal name/address/country consistent across Play Console, payments profile, and business documents.

2) Current project build state

  • targetSdkVersion: 36 (meets Android 15+ requirement)
  • compileSdk: 36
  • minSdk: 24
  • versionCode: 20
  • versionName: 12.0.1
  • Google Mobile Ads SDK: com.google.android.gms:play-services-ads:25.0.0
  • Firebase Analytics enabled
  • Release build uses R8 minify + resource shrinking

3) Items that must be completed directly in Play Console

A. Data safety form (required)

  • Missing form blocks updates/publication.
  • Because Firebase Analytics + AdMob are used, do not submit as "No data collected".
  • Verify and declare data collection/processing, purposes (analytics/ads), advertising ID usage, encryption in transit, and deletion request channel.

B. Privacy policy URL (Store listing)

  • Must be publicly reachable.
  • Must match actual app behavior (analytics + ads + network fetch).

C. Publish a new production release

  • Old production warning (versionCode 18, targetSdk 29) is cleared only after a newer compliant production release is published.
  • Internal/closed testing alone may not clear production availability warnings.

4) Suggested rollout

  1. Internal testing track
  2. Closed testing track
  3. Production staged rollout (5% -> 20% -> 100%)

5) Post-release checks

  • Play policy status: Target API warning cleared, outdated GMA SDK warning cleared, data safety issue cleared.
  • Firebase events visible in DebugView/Analytics dashboard.
